# コーディング指示マニュアル

AI に正確・効率的に指示するためのガイドです。

## 基本原則

- シンプルに、最短で伝える
- 「どこを」「どうする」を明確に
- 複雑な手順より素直な方法を選ぶ

## 1. 要素の配置・位置調整

```
[クラス名] を [基準となる親クラス名] に合わせて、top: [値], right: [値] に設置して
```

例: `p-careerpathSteps__decoSvg` を `p-careerpathSteps__body` に合わせて `top: -131px` など。

## 2. レイアウトの制約

```
[クラス名] がインナーに合わせて、無限に広がらないようにして
```

## 3. 要素の順番変更

```
[クラス名] の [N番目] と [M番目] の順番が逆
```

## 4. 見た目の修正

```
[クラス名] ここが [現状] なので、[期待する状態] にして
```

## 5. 不足パーツの追加

```
[ページ名]に[パーツ名]がない（他ページの ○○ と同じ）
```

## 6. Figma からの実装

```
[対象クラス名] を Figma から実装して。[Figma URL]
（z-index や「背面に回す」などの意図があれば併記）
```

## 指示の 3 要素

| 要素 | 説明 |
|------|------|
| **どこ** | クラス名 |
| **基準** | 親要素・参考ページ |
| **どうする** | 具体的な値・状態 |
